System Walkthrough

MU'UJIZA LIS

A centralized Laboratory Information System built for small-to-medium diagnostic labs — unifying all units on a single platform.

⚗️ Multi-Unit 💳 Integrated Payments 📶 Offline-First 🔐 Role-Based Access 📄 Single Result Sheet

What Makes This Different

Most lab systems are department-isolated — Haematology has their register, Microbiology has theirs, Chemical Pathology has theirs. A patient with 3 tests visits 3 counters and collects 3 separate papers.

MU'UJIZA merges all units into one sample record. Every department enters their results into the same sample. The patient collects one result sheet from one counter — regardless of how many units processed their sample.

Sample Journey — End to End
1
Reception

Sample Registration

Patient arrives. Reception enters demographics, selects tests from across all lab units, and processes payment (cash or Paystack online). Half-payment accepted — balance tracked. Receipt printed immediately. Sample gets a unique MU-#### ID.

2
Technologist

Sample Assessment

Technologist opens the sample in Result Entry. If the sample is unsuitable (haemolysed, wrong container, insufficient volume), they reject it with a documented reason — status changes to Rejected and is visible immediately on Track Sample. Otherwise, processing begins.

3
Technologist

Result Entry

Each technologist enters results for their assigned unit only. The system auto-flags abnormal values against age- and gender-matched reference ranges. When all tests in the unit are marked Ready, the sample automatically moves to verification queue. Multiple units can work simultaneously on the same sample.

4
Supervisor / HOD

Verification & Release

Supervisor reviews all results, checks payment status, and may add a clinical comment. If a result needs correction, it is returned to the technologist with a query — this is audited. Once satisfied, the supervisor authorises and releases the result for collection.

5
Patient Portal

Result Collection & Printing

Counter staff search the sample by ID, name, or phone. Released results appear — all units consolidated on one page. Staff print a single result sheet covering every test ordered regardless of which department processed it. Outstanding balance can be settled here before printing.

Offline Resilience

📶 Works Without Internet — With One Important Rule

Login requires an active connection — this ensures only registered staff access the system and all cached data is fresh and synced on entry.

Once logged in, all result entry, sample viewing, and draft saving works completely offline. Any changes made offline are queued in an encrypted local outbox (IndexedDB) and automatically synced to the server the moment connectivity returns — in the exact order they were made.

A live status banner shows staff whether they are online, offline, or syncing pending changes. No data is ever lost.
